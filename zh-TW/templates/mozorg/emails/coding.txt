== Firefox 除錯 ==

首先，Firefox 的基礎架構是 C/C++ / JS / XUL，只要了解 JS 或 CSS 就能幫 Firefox 除錯，在 Mozilla 寫軟體有蠻多面向可以選擇，例如你可以學習 Firefox 或 Thunderbird 的程式設計，主要以 C 或 C++ 為主。其實 Firefox Coding 並不難，對於剛起步的應該可以先從跑流程開始。兩個較大的挑戰是 build Firefox 與 hg version control system。

您可以先參考以下介紹，了解基本概念：
Mozilla Links 正體中文版: 摩茲動手做（一）如何幫 Firefox 抓蟲 http://mozlinks-zh.blogspot.com/2012/02/firefox_25.html

我們社群目前主打 Good First Bug 專案，一步步協助大家嘗試解決 Firefox 的第一個 bug，這是最近我們整理出來的新手 bug 列表：https://etherpad.mozilla.org/moztw-nctu-service-2013 你可以選擇有興趣的 Bug 挑戰。

你還可以上 Bugs Ahoy，查詢你的專長方面有哪些新手 Bug 可以解：http://www.joshmatthews.net/bugsahoy/ 先看看你對什麼比較有興趣，我們可以告訴你更深入的資訊


==Firefox OS==

Firefox OS 主要有三個層次 Gonk, Gecko, Gaia：
* Gonk: 作業系統相關，如果你有興趣 porting Firefox OS 到新裝置請參考相關文件[1]
* Gecko: 包含 Web API 實作以及 Browser Engine，此部份跟一般 Firefox Browser 開發相同，所作的變更也是與 Forefox Browser 共用（只是有可能預設關閉）
* Gaia: Firefox OS 的預載 App，如 Calendar、Gallery、Music…等等，也包含 Notification 以及 Windows Manager 等系統元件。文件請直接參考 github README[2]

如果你對在 Firefox OS 上面開發程式（不包含上面所提的 Gaia 預載 apps）感興趣，請參考 Marketplace Developer Hub 中的文件[3]。

如果您想參與 Firefox OS QA，請看 irc.mozilla.org #fxosqa 頻道。

[1] https://developer.mozilla.org/en-US/docs/Mozilla/Firefox_OS/Porting
[2] https://github.com/mozilla-b2g/gaia
[3] https://marketplace.firefox.com/developers/
