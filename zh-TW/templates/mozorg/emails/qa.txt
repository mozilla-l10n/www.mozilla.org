== QA 測試工作 方面 ==

Mozilla 的 QA 工作，最簡單的貢獻方法是在日常生活中使用全系列的 Nightly 版 Firefox，並在發現問題時到 Bugzilla 回報 Bug。
  - Firefox Nightly 下載： http://nightly.mozilla.org/
  - Firefox 的開發頻道相關說明： http://mozlinks-zh.blogspot.com/2012/12/firefox_5.html
  - Bugzilla 的教學影片： http://vimeo.com/9205730
  - Bugzilla 的投影片： https://wiki.mozilla.org/images/6/6d/IntroductionToBugzilla.pdf


Mozilla QA 團隊的主要聯絡管道（英文為主）為 IRC 與郵件群組：
  - 您可以透過各團隊的 IRC 頻道與大家線上對談，IRC 的詳細說明請看： https://wiki.mozilla.org/IRC
  - 當你加入郵件群組後，就會開始收到群組中其他人的信件，也可以回信與他人共同討論。加入時，建議先發一封信件自我介紹一下： http://groups.google.com/group/mozilla.dev.quality/


Mozilla QA 的專案平台為 QMO，上面提供各專案 QA 相關資訊，請先閱讀，並可按照說明直接聯繫、加入相關管道以參與： https://quality.mozilla.org/docs/misc/how-can-i-help-test/


=== One and Done ===

QA 團隊的 One and Done 系統，可以讓你按照擁有的空閒時間，提供各種不同的任務讓你執行。歡迎在有空時登入 One and Done 並且按照說明解任務： https://oneanddone.mozilla.org/
（如果在測試中途發現問題時，請記得寫信（英文）給到每個任務的負責人喔） 


以下是 Mozilla 各產品的個別 QA 資訊與管道


=== Desktop Firefox ===
  - 安裝 Nightly 測試版，並協助回報 Bug： http://nightly.mozilla.org/
  - 安裝 Test Pilot 並參與不定期使用者問卷： https://testpilot.mozillalabs.com/
  - 參加每週五的全球 Testday： https://quality.mozilla.org/tag/test-day/
  - 加入 IRC 的 #qa 頻道
  - 追蹤 dev-quality 郵件群組： https://lists.mozilla.org/listinfo/dev-quality


=== Android Firefox ===
  - 使用 Nightly 版、協助回報 Bug： https://quality.mozilla.org/docs/mobile-firefox/how-to-install-fennec-nightly-builds/
  - 追蹤 QA 郵件群組： https://mail.mozilla.org/listinfo/android-mobile-nightly-testers


=== Firefox OS ===
  - 加入 B2G QA 郵件群組： https://wiki.mozilla.org/B2G/QA
  - IRC #appsqa、#fxosqa 頻道


=== 網頁 WebQA ===
  - 掛 IRC #mozwebqa
  - 作 exploratory testing： https://quality.mozilla.org/2012/11/exploratory-testing-on-moztrap-job-board-posting-1/


=== Firefox 套件 ===
  - 安裝 Add-on Compatibility Reporter 套件，協助回報與新版 Firefox 不相容的套件： https://addons.mozilla.org/zh-tw/firefox/addon/add-on-compatibility-reporter/
  - 如果您有套件的開發經驗，可以加入 Addon Editors 協助審核跟編輯套件精選： https://wiki.mozilla.org/AMO:Editors


另外，當您有空也可以幫忙看看 Firefox、Thunderbird 討論區，發現使用者有發現什麼 Bug，就協助報上 Bugzilla。
